# Plan: Decouple Rust emit-specific imports from pytra-cli2.py (P0-CLI2-RS-DECOUPLE)

## Background

`pytra-cli2.py` has already switched the C++ emit path to subprocess delegation, but the Rust emit path still depends at the top level on `toolchain2.emit.rs.emitter` and `toolchain2.link.manifest_loader`.

As a result, when `src/pytra-cli2.py` is transpiled for C++ selfhost, the unnecessary Rust emitter and its dependencies enter the include graph, pulling in headers such as `shutil.h` that cannot be compiled by C++.

## Current State

- `src/pytra-cli2.py` has `from toolchain2.emit.rs.emitter import emit_rs_module`
- `src/pytra-cli2.py` has `from toolchain2.link.manifest_loader import load_linked_output`
- `_emit_rs()` / `_write_rs_package_files()` / `_copy_rs_runtime_files()` / `_manifest_type_id_table()` / `_generate_type_id_table_rs()` remain in `pytra-cli2.py`

## Approach

- Move Rust emit to `toolchain2.emit.rs.cli`, the same as C++
- `pytra-cli2.py` will only call `python3 -m toolchain2.emit.rs.cli ...` as a subprocess
- The behavior of `-emit --target rs` and `-build --target rs` is preserved

## Specific Changes

1. Add `src/toolchain2/emit/rs/cli.py` and move manifest loading, package emit, runtime file copy, and type-id table generation into it
2. Remove the top-level Rust emitter / manifest loader imports from `src/pytra-cli2.py`
3. Change the Rust emit/build path in `src/pytra-cli2.py` to subprocess delegation
4. Add a regression test to `tools/unittest/tooling/test_pytra_cli2.py` to verify that Rust emit imports no longer flow in

## Acceptance Criteria

- Rust emit / manifest loader are gone from `pytra-cli2.py`'s top-level imports
- `pytra-cli2.py -emit --target rs ...` and `-build --target rs ...` pass
- `toolchain2/emit/rs/emitter.h` is not included in the `pytra_cli2.h` generated for C++ selfhost

## Completion Notes

- Added `src/toolchain2/emit/rs/cli.py` and separated Rust emit's manifest loading, package output, and runtime copy from `pytra-cli2.py`
- `src/pytra-cli2.py` now delegates Rust emit to `python3 -m toolchain2.emit.rs.cli` as a subprocess, and the top-level Rust emitter / manifest loader imports have been removed
- `PYTHONPATH=src python3 src/pytra-cli2.py -build sample/py/17_monte_carlo_pi.py --target rs --rs-package -o work/tmp/cli2_rs_emit` succeeded
- The `work/selfhost/build/cpp/emit/pytra_cli2.h` generated by `PYTHONPATH=src python3 src/pytra-cli2.py -build src/pytra-cli2.py --target cpp -o work/selfhost/build/cpp/emit` contains neither `toolchain2/emit/rs/emitter.h` nor `shutil.h`
